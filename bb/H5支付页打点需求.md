# H5支付页打点需求

## 一、需求背景

12/27 上线「PWA支付流程优化」后，促销弹窗和高荷页的支付入口改为跳转H5支付页。由于H5支付页是服务端渲染，原前端的 `payment_continue_click` 打点丢失，导致漏斗数据缺失。

**影响范围**：
- 促销弹窗订单数：12/27起数据为0
- 高荷页订单数：12/27起数据为0

## 二、需求描述

在H5支付页点击「继续支付」按钮时，由H5支付页前端上报 `payment_continue_click` 事件，事件格式与原PWA前端打点保持一致。

**说明**：H5支付页由服务端团队维护，PWA前端无法直接修改，需服务端团队协助完成打点。

## 三、打点规范

### 事件名称
`payment_continue_click`

### 触发时机
用户在H5支付页点击「继续支付」/「Pay Now」按钮时

### 事件字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| event | string | 是 | 固定值：`payment_continue_click` |
| user_id | string | 是 | 用户ID |
| extra | json string | 是 | 扩展字段，见下表 |

### extra 字段详情

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| entry_source | string | 是 | 入口来源，用于区分订单类型 |
| country | string | 否 | 国家代码，如 "USA" |
| total_amount | string | 是 | 订单金额 |
| user_tier | string | 否 | 用户等级，如 "guest" |
| device_type | string | 否 | 设备类型，如 "ios"/"android" |
| selected_package | json string | 否 | 选中的商品信息 |
| selected_method | json string | 否 | 选中的支付方式 |
| previous_balance | string | 否 | 用户当前余额 |

### entry_source 枚举值

| 值 | 说明 | 对应漏斗指标 |
| --- | --- | --- |
| `pop_promotion` | 促销弹窗入口 | 促销弹窗订单数 |
| `sexy_teaser_page` | 高荷页入口 | 高荷页订单数 |

## 五、验收标准

- [ ] H5支付页点击「继续支付」按钮时，上报 `payment_continue_click` 事件
- [ ] `entry_source` 字段正确区分促销弹窗（`pop_promotion`）和高荷页（`sexy_teaser_page`）
- [ ] 事件数据可在数据平台查询到
- [ ] 漏斗报表中「促销弹窗订单数」「高荷页订单数」恢复正常

## 六、优先级

**P0 - 紧急**

数据缺失直接影响漏斗分析和转化率统计。

## 七、期望上线时间

2025/1/3 前
