# 全流程 SOP

> 版本：v1.0 | 2026年3月9日
> 适用范围：iOS 马甲包从账号准备到上架的完整链路

---

## 一、目标与数据看板

### 核心 KPI（周维度）

| 指标 | 说明 | 健康基准 |
|------|------|---------|
| 提审数量 | A面 + B面分开统计 | A面 ≥ 2 / B面 ≥ 3 |
| 过审率 | 过审数 ÷ 提审数 | ≥ 90% |
| 卡审数量 | 超过 3 天未出结果 | 0 |
| 拒审数量 | 含原因分类 | 越少越好 |
| 封号数量 | 含 4.3 下架 | 0 |
| 可用账号库存 | 当前可用账号数 | ≥ 10 个 |

### 每周数据复盘节奏

- 周一早上：看上周数据，识别异常
- 周五下午：更新本周数据，写入周报

## 二、阶段一：前期准备

### 账号购买标准

| 维度 | 标准 |
|------|------|
| 供应商 | 美国新供应商优先，禁用养号历史列表 |
| 账号状态 | 未使用过的新号，无提审历史 |
| 国家分配 | 新 App 账号分散不同国家，不集中同一地区 |
| 养号 | 暂停，效果不明显且影响迭代节奏 |

### IP 购买规范

| 维度 | 标准 |
|------|------|
| 类型 | 静态住宅 IP，禁用机房 IP |
| 地区 | 优先欧洲 |
| 隔离原则 | 一个 IP 只绑定一个账号，严禁复用 |
| 污染判断 | 曾出现卡审/封号记录的 IP 视为污染，立即停用 |

### IP 购买步骤（iproyal）

1. 登录 [iproyal 控制台](https://dashboard.iproyal.com/)
2. 选择「静态住宅代理」
3. 创建新订单：
   - 周期选 **30天**
   - 数量 **≥ 2 个**（2个起购更划算）
   - 地区选 **欧洲**
   - **必须勾选「质量」选项**
4. 提交订单，获得 socks5 账号密码
5. 在苹果账号管理表中填写以下信息：
   - 网络（socks5 账号密码）
   - 账号内容（Apple ID 账号密码）
   - IP 过期时间
   - IP 是否自动续费

### 账号管理表字段

每个账号必须记录：IP 地址 / socks5账号密码 / Apple ID账号密码 / IP过期时间 / IP自动续费状态 / 当前状态 / 绑定包名 / 异常备注

## 三、阶段二：装机

> 参考耗时：50 分钟/台
> 适用环境：Windows 宿主机 + VMware 虚拟机（macOS）

### 装机步骤

1. 根据磁盘大小在 VMware 中**创建新虚拟机**
   - 虚拟机名称示例：`YSYS`、`YSVM`（与账号国家对应，便于管理）
   - 位置选择剩余空间充足的磁盘

2. **自定义硬件配置**
   - 内存：**8 GB**
   - 处理器：**6 核**
   - 网络适配器：改为**启动时关闭**（避免装机过程联网）

3. **开启虚拟机，格式化磁盘**
   - 进入安装界面后选择 **Disk Utility**
   - 点击 **Erase**，将磁盘名称改为 `YSSHDM`
   - 完成后关闭 Disk Utility，继续安装

4. **安装 macOS 系统**（等待安装完成）

5. **选择国家**：与账号国家保持一致，后续步骤一直下一步

6. **创建系统账号**
   - 用户名与虚拟机名称保持一致
   - 位置（地区）与账号国家保持一致

7. **关机，安装 VMware Tools**
   - 在 VMware 菜单中选择安装 VMware Tools
   - 安装完成后重启虚拟机

8. **获取环境安装包**
   - 从 FTP 中转目录下载 `jgaurora.zip`
   - 将文件从宿主机拖入虚拟机

9. **执行环境安装脚本**
   - 解压 `jgaurora.zip`
   - 运行 `install_flutter_env.sh` 脚本，等待安装完成

## 四、阶段三：验号

> 参考耗时：20 分钟/个

### 验号步骤

1. 本机先连接**快连**（用于访问外部网络环境）
2. 选择一台对应国家、已装机的机器并启动
3. 打开 **Clash Verge**（装机时已预装）
4. 打开 **ip.yaml** 文件，将购买的静态 IP 相关信息替换进去
5. 将 ip.yaml 文件拖入 Clash Verge，并做以下配置：
   - 开启 **Tun 模式**
   - 开启 **Auto Launch**
   - proxies 选 **Global**
   - 选择对应网络
6. 打开 [ip.sb](https://ip.sb) 确认当前 IP 与购买的静态 IP 一致
7. 登录 [account.apple.com](https://account.apple.com) 修改苹果账号密码
8. 登录开发者账号（注意：目前不登录设备，需关注接码过期时间）
9. 在苹果账号管理表中更新以下字段：
   - 验号时间
   - 续费时间
   - 注册时间
   - 备注
   - 账号内容
   - 账号状态改为**「待使用」**

> 云便签（远程同步文档）：[webnote.cc](https://webnote.cc/)

### 验号结果判断

| 结果 | 判断标准 | 处理 |
|------|---------|------|
| 通过 | IP 一致、登录正常、密码已修改 | 状态改为「待使用」，进入开发循环 |
| 异常 | 登录失败、账号锁定、IP 不一致 | 更换账号或 IP，重新验号 |
| 降级 | 账号有历史提审记录且曾卡审 | 降级使用或弃用 |

## 四、阶段三：开发循环

> 最大耗时环节：A1 开发（1~1.5 天）

### 开发顺序

A1 → A2 → H5B，不等 A1 过审再开始 A2，并行推进

### 发包前验收 Checklist

**A 面**
- [ ] A1 UI 还原度达标，无明显抄袭特征
- [ ] A2 内购功能配置正确，内购主路径验证通过
- [ ] 代码混淆已执行
- [ ] 第三方库已替换/优化

**资产**
- [ ] Logo 已到位
- [ ] 五图内容与 App 功能匹配
- [ ] 应用名称无重复

**提审资料**
- [ ] 银行卡已绑定
- [ ] App 描述完整，语言与目标市场匹配
- [ ] 隐私政策链接有效
- [ ] 年龄分级填写正确

## 五、阶段四：提审与发布

> 提审流程本身约 1 小时

### 并行策略

单账号审核等待期间，同步推进其他账号开发，不串行等待

### 审核结果处理

| 结果 | 处理方式 |
|------|---------|
| 过审 | 发布应用，更新投放包表，通知投放侧 |
| 卡审（>3天） | 撤审，更换账号重提，不在原账号继续等待 |
| 拒审 | 24小时内完成原因分析，48小时内给出修复方案，见下节 |

## 六、拒审处理

### 拒审原因分类与修复

| 类型 | 典型表现 | 修复方向 |
|------|---------|---------|
| 4.3 设计相似 | 图标/界面与已有 App 高度相似 | 调整 Logo 色系/图形，修改主界面布局，更换应用名称 |
| 代码重复 | 使用相同第三方库或代码结构 | 替换/封装第三方库，加强代码混淆 |
| 内购问题 | 内购路径不完整、定价异常 | 验证内购主路径完整性，重新配置内购项目 |
| 内容违规 | 描述/截图含违规内容 | 修改描述和截图 |
| 资料不完整 | 隐私政策、年龄分级缺失 | 补全资料后重提 |

### 拒审记录（每次必填）

处理完后录入 [04_问题知识库.md](04_问题知识库.md)：

```
包名：

拒审原因（Apple 原文）：
原因分类：
修复方案：
重提时间：
重提结果：
```

## 七、封号处理

### 即时动作（1 小时内）

- [ ] 确认封号类型（账号封禁 / 4.3 下架 / 违规下架）
- [ ] 评估该账号下在投放的包，通知投放侧切换
- [ ] 账号管理表状态更新为"封号"，记录时间和原因
- [ ] 检查同批次账号（同供应商、同 IP 段）是否有连带风险

### 根因分析（24 小时内完成，录入知识库）

```
封号账号：
封号时间：
封号原因（已知/推测）：
该账号 IP 情况：
该账号发包历史：
关联账号是否受影响：
根因判断：□ 账号质量  □ IP 污染  □ 内容违规  □ 其他
改进措施：
```

### 账号池补充触发机制

| 触发条件 | 动作 |
|---------|------|
| 可用账号 < 5 个 | 立即补充采购 |
| 连续封号 ≥ 3 个 | 暂停当前供应商，排查 IP 和账号质量 |
| 卡审率 > 30% | 全面审查账号来源和 IP 配置 |
