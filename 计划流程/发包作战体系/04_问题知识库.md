## 三、卡审案例

**[2026-02-01 周] 越南/印度国家批次**
- 现象：有一次审核问题后，再次提交直接卡审（等待审核状态）
- 根因：账号或 IP 被标记，重提后直接进入人工审核队列

## 四、经验总结

### 账号策略

- 养号无效，反而影响迭代节奏（2026-02 验证）
- 美国新供应商账号卡审率明显低于其他来源
- 新 App 账号要分散不同国家，不能集中在同一地区

### IP 策略

- 住宅 IP 优先，机房 IP 容易被识别
- 一个 IP 只绑一个账号，复用必然出问题
- 曾出现卡审/封号的 IP 直接停用，不要侥幸

### 发包节奏

- A1 → A2 → H5B 并行推进，不串行等待
- 卡审超过 3 天直接撤审换号，不要继续等
- 发包工具全面使用后，本周（0302-0308）未出现卡审，效果明显

### 过审规律

- A1 审核速度相对 A2+H5 快
- 过审情况下优先以 A1+A2+H5+五图方式开发
- 目标：达到 2-3 小时过审（静界组已有 1 个达到）
