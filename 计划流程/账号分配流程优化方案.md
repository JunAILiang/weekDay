# 账号分配流程优化方案

> 版本：v3.0 | 2026年3月31日
> 背景：账号被过早锁定，导致可用账号不足、发包人员状态混乱

---

## 一、根因分析

### 当前问题流程

```
开始开发 → 占用账号 → 等五图 → 等代码完成 → 提审
         ↑
    账号在此锁死，可能空转 2~3 天
```

账号是稀缺资源，但当前在**开发阶段就被分配**，导致：

- 五图未到、代码未完成期间，账号处于空转占用状态
- 真正准备好提审的包拿不到可用账号
- 多人同时开发时，账号池迅速耗尽

### 正确应有的流程

```
开发代码 ──┐
申请五图 ──┼─→ 全部 Ready → 申请账号 → 提审
备齐资料 ──┘
                             ↑
                     账号应在此才分配
```

**核心原则：账号是提审工具，不是开发工具，应在最后一步才分配。**

---

## 二、解决方案

### 2.1 开发阶段拆成两条并行线

代码开发与资产申请不再串行，同步推进：

```
主线（代码）：  A1 开发 → A2 开发 → H5B 开发 → 完成
并行线（资产）：同步提 Logo + 五图需求
```

代码完成后，根据资产状态走不同路径：

| 情况 | 处理方式 |
|------|----------|
| Logo + 五图已到位 | 直接进入提审流程 |
| Logo + 五图还需 2 天+ | 用 AI 生成 Logo + 截图代替五图，先提审；过审后再补提正式五图需求 |

> **原则：不让资产等待阻塞提审节奏，先用「够用」的素材过审，正式素材过审后补。**

---

### 2.2 申请账号的前置条件

以下条件全部满足后，方可申请账号：

| # | 前置条件 |
|---|----------|
| 1 | A1 + A2 + H5B 代码完成，本地打包验证通过 |
| 2 | Logo 已到位（正式或 AI 生成临时版均可） |
| 3 | 五图已到位（正式或截图临时版均可） |
| 4 | 应用名称、描述文案已定稿 |
| 5 | 隐私政策链接有效可访问 |

> 任意一条未满足，不得申请账号。

---

### 2.3 包状态机调整

在包进度管理表中，将状态拆细：

```
开发中 → 资产待齐 → Ready → 账号已分配 → 提审中 → 结果处理
                      ↑
              5 项条件全部满足后才能进入此状态
              账号负责人看表分配，不看口头申请
```

| 状态 | 是否可申请账号 | 说明 |
|------|---------------|------|
| 开发中 | ❌ 不可 | 代码或资产任意一项未完成 |
| 资产待齐 | ❌ 不可 | 代码完成，资产仍在等待 |
| **Ready** | ✅ **可申请** | 5 项条件全部满足（含临时素材） |
| 账号已分配 | — | 已分配，24h 内必须完成提审，否则释放账号 |
