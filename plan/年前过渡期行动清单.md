# 年前过渡期行动清单（1周版）

> 适用场景：只有1周时间 + 还要兼顾PWA交接
> 核心目标：建立初步认知和信任，年后全力接手

---

## 🎯 这一周的核心目标

**不是**：全面接手团队
**而是**：
1. ✅ 让团队知道你的存在和风格（建立信任）
2. ✅ 搞清楚最紧急的问题是什么（审核问题）
3. ✅ 拿到必要的权限和数据（年后能立即开工）
4. ✅ 让主管放心（你在认真对待）

**什么可以等到年后：**
- ❌ 深入了解所有技术细节
- ❌ 亲自走一遍完整流程
- ❌ 制定30天行动计划
- ❌ 启动Quick Win项目
- ❌ 优化流程和工具

---

## 📅 每天要做的事（精简版）

### Day 1（明天）- 最重要的一天 ⭐⭐⭐⭐⭐

#### 上午（2小时）

**Todo 1：参加早会（30分钟）**
- [ ] 按照我们之前讨论的方式介绍自己
- [ ] 重点：多听少说，了解团队氛围
- [ ] 记录：团队现在最关心什么问题

**Todo 2：与主管快速对齐（1小时）**

只问最关键的3个问题：

1. **审核问题的紧急程度**
   - [ ] 现在拒审率是多少？
   - [ ] 最近一次过审是什么时候？
   - [ ] 年前这一周有什么必须做的吗？

2. **年前这一周的期望**
   - [ ] 主管期望我这一周做什么？
   - [ ] 什么可以等到年后？

3. **权限和资源**
   - [ ] 我需要哪些权限？（代码、数据、账号）
   - [ ] 谁能帮我开通？

#### 下午（1小时）

**Todo 3：获取所有必要权限（1小时）**
- [ ] 代码仓库访问权限
- [ ] 数据看板访问权限（如果有）
- [ ] 账号管理系统访问权限
- [ ] 文档访问权限
- [ ] 测试能否访问

**💡 为什么第一天只做这些？**
- 你还有PWA交接工作
- 不要给团队太大压力
- 先建立存在感

---

### Day 2-3（接下来2天）- 建立初步认知

#### 每天上午（1小时）

**Todo 1：参加早会（15分钟）**
- [ ] 听团队成员说昨天做了什么、今天做什么
- [ ] 记录：有什么问题？谁在做什么？
- [ ] 不要急着做决策，先观察

**Todo 2：快速了解数据（30分钟）**
- [ ] 看最近7天的流水数据
- [ ] 看最近的过审情况
- [ ] 记录：最明显的3个问题是什么？

**Todo 3：与1-2个核心成员聊天（30分钟）**

Day 2：找Flutter开发A（最强的那个）
- [ ] 问：现在最大的问题是什么？
- [ ] 问：审核为什么被拒？
- [ ] 问：有什么建议？

Day 3：找Kotlin开发
- [ ] 问：方向2现在什么情况？
- [ ] 问：有什么困难？
- [ ] 问：需要什么支持？

#### 下午（专注PWA交接）

这两天下午专注做PWA交接，不要分心。

---

### Day 4（倒数第二天）- 梳理信息

#### 上午（2小时）

**Todo 1：参加早会（15分钟）**

**Todo 2：整理这几天收集的信息（1.5小时）**

创建一个简单的文档，记录：

**1. 审核问题**
- 拒审率：XX%
- 主要原因：XXX
- 最近一次过审：XX月XX日

**2. 产能情况**
- 现在一周产出：XX个包
- 目标产能：XX个包
- 最大瓶颈：XXX

**3. 团队情况**
- Flutter开发A：技能水平、在做什么、有什么问题
- Flutter开发B：技能水平、在做什么、有什么问题
- Kotlin开发：技能水平、在做什么、有什么问题

**4. 年后要做的Top 3事情**
- 第1件事：XXX
- 第2件事：XXX
- 第3件事：XXX

#### 下午（专注PWA交接）

---

### Day 5（最后一天）- 向上汇报

#### 上午（1.5小时）

**Todo 1：参加早会（15分钟）**

**Todo 2：与主管汇报（1小时）**

汇报内容：

1. **这一周我做了什么**
   - 参加了早会，了解了团队
   - 收集了基本信息
   - 拿到了必要权限

2. **我看到的问题**（用数据说话）
   - 审核问题：拒审率XX%，主要原因是XXX
   - 产能问题：现在XX个/周，目标XX个/周
   - 团队问题：XXX

3. **年后的计划**
   - 第一周：深入了解方向1，亲自走一遍流程
   - 第二周：制定30天行动计划
   - 第一个月：解决审核问题，提升产能

4. **需要的支持**
   - 年后需要XXX资源
   - 需要主管帮忙协调XXX

**💡 为什么要汇报？**
- 让主管知道你在认真对待
- 管理期望
- 年后能立即获得支持

#### 下午（专注PWA交接）

## ⚠️ 这一周的注意事项

### ✅ 要做的事

1. **多听少说**
   - 这一周主要是观察和了解
   - 不要急着做决策
   - 不要急着改变什么

2. **建立存在感**
   - 每天参加早会
   - 让团队知道你在关注
   - 但不要给压力

3. **拿到权限**
   - 这是最重要的
   - 没有权限年后什么都做不了

4. **向上汇报**
   - 让主管知道你在做什么
   - 管理期望

### ❌ 不要做的事

1. **不要陷入执行细节**
   - 不要亲自去改代码
   - 不要亲自去打包
   - 这一周你还有PWA交接

2. **不要急着做决策**
   - 不要急着改流程
   - 不要急着定方案
   - 信息还不够

3. **不要给团队太大压力**
   - 不要一上来就提很多要求
   - 不要批评现有做法
   - 先建立信任

4. **不要过度承诺**
   - 不要说"我一定能解决XXX"
   - 说"我会认真了解，年后我们一起解决"
