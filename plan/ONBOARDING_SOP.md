# 新任研发主管接手SOP

> 创建时间：2026-02-04
> 用途：Flutter马甲包团队接手指南

## 📋 文档概述

本文档是新任研发主管接手Flutter马甲包团队的标准操作流程（SOP），包含需要收集的信息清单、优先级判断和行动计划。

## 🎯 团队基本情况

### 人员配置
- **总人数**：5人（包括你自己）
- **Kotlin开发**：1人（只会Android Kotlin）
- **Flutter开发**：2人
- **你自己**：Flutter背景
- **待招聘**：1人（Flutter + Android）

### 三个业务方向

**方向1：iOS Flutter包（当前最优先）**
- 技术架构：A1（A面）+ A2（内购）+ H5（webview）
- 当前状态：由主管负责，即将交接
- 主要问题：产能不够、效率慢、遭遇审核拒审
- 重要性：⭐⭐⭐⭐⭐（现金流来源，必须保住）

**方向2：Android Kotlin印度包**
- 技术架构：原生Kotlin，滚包方式
- 当前状态：主管跟进中，后续会交接
- 重要性：⭐⭐⭐（主管关注，证明能力的机会）

**方向3：Flutter Android全球包**
- 当前状态：投放投不起来，暂时搁置
- 重要性：⭐⭐（有潜力，但暂时不是重点）

## 🚀 第一天/第一周行动清单

### 第一天（Day 1）

#### 上午：建立基础认知
1. **与主管深度沟通（2-3小时）**
   - 了解方向1的紧急问题（特别是审核拒审情况）
   - 明确你的KPI和考核标准
   - 了解公司对三个方向的优先级
   - 确认交接时间表和支持力度

2. **获取必要权限**
   - 代码仓库访问权限（GitLab/GitHub）
   - 数据看板访问权限
   - 账号管理系统访问权限
   - 相关文档和知识库访问权限

#### 下午：团队见面和数据了解
3. **与团队成员1对1沟通（每人30分钟）**
   - 了解每个人的技能水平
   - 了解他们当前在做什么
   - 了解他们对项目的看法和痛点

4. **查看当前数据**
   - 查看最近的流水数据
   - 查看最近的过审率/拒审率
   - 查看当前的产能数据

### 第一周（Week 1）

#### Day 2-3：深入了解方向1
- 完整走一遍iOS包的制作流程
- 亲自操作一次打包流程
- 记录每个步骤的耗时和痛点
- 分析最近被拒审的案例

#### Day 4-5：制定初步方案
- 识别最紧急的问题（审核问题优先）
- 制定短期应对方案（1-2周内）
- 规划中期优化方向（1-2个月）
- 与主管同步你的想法

## 🎯 战略建议和优先级

### 精力分配建议（前3个月）

**你自己的精力分配：**
- **40%** - 方向1（iOS Flutter）：把控质量、解决审核问题、增长策略
- **30%** - 团队管理和提效工具：建立流程、优化效率
- **20%** - 方向2（Android Kotlin）：准备接手、了解情况
- **10%** - 方向3（Flutter Android）：战略规划、保持关注

### 人员分配建议

**你自己：**
- 前2周：深度了解方向1，亲自操作流程
- 第3-4周：制定优化方案，开始解决审核问题
- 第2-3个月：逐步接手方向2，同时优化方向1

**Kotlin开发：**
- 短期内100%投入方向2
- 不要分散精力到其他方向
- 等新人到位后可以考虑让他学习Flutter

**Flutter开发A（最强）：**
- 80%投入方向1（主力维护和产出）
- 20%参与提效工具开发

**Flutter开发B：**
- 60%投入方向1（辅助）
- 40%探索方向3（为未来做准备）

**未来的Flutter+Android开发：**
- 到位后50%支持方向2（分担Kotlin开发压力）
- 50%支持方向1或方向3（根据当时情况）

## ⚠️ 风险识别和应对

### 高风险项

**风险1：审核问题持续恶化**
- **影响**：流水断崖式下跌，影响公司收入
- **应对**：
  - 第一周内必须搞清楚拒审原因
  - 准备多套应对方案（技术方案、马甲化方案）
  - 必要时寻求外部专家支持
  - 建立应急预案

**风险2：Kotlin开发是单点风险**
- **影响**：方向2完全依赖一个人，他离职或生病会导致项目停摆
- **应对**：
  - 尽快让新人能接手方向2
  - 你自己也要尽快熟悉方向2
  - 建立完善的文档
  - 考虑让Flutter开发也学习Android

**风险3：产能提升不及预期**
- **影响**：无法完成KPI，影响团队士气
- **应对**：
  - 先优化最大的瓶颈
  - 不要试图同时优化所有环节
  - 设定合理的阶段性目标
  - 及时向上管理期望

**风险4：团队成员能力不足**
- **影响**：无法独立完成任务，你成为瓶颈
- **应对**：
  - 快速评估每个人的真实水平
  - 制定培训计划
  - 必要时申请更换人员或增加人手
  - 建立导师制度

## 🚀 快速见效的优化方向

### 第一个月可以做的（Quick Wins）

**1. 建立数据监控看板（优先级：⭐⭐⭐⭐⭐）**
- 如果没有看板，立即建一个
- 至少包含：流水、过审率、产能、账号状态
- 让团队每天都能看到数据
- **预期效果**：提升团队对问题的感知能力

**2. 整理和优化混淆脚本（优先级：⭐⭐⭐⭐⭐）**
- 统一管理所有混淆脚本
- 建立混淆规则库
- 一键生成混淆代码
- **预期效果**：减少50%的手工操作时间

**3. 建立每日站会机制（优先级：⭐⭐⭐⭐）**
- 每天15分钟，同步进度和问题
- 快速响应问题
- **预期效果**：提升团队协作效率

**4. 梳理账号管理（优先级：⭐⭐⭐⭐）**
- 清点所有账号
- 评估账号健康度
- 建立账号轮换机制
- **预期效果**：降低账号被封风险

### 第2-3个月可以做的（中期优化）

**5. 打包流程自动化（优先级：⭐⭐⭐⭐⭐）**
- 建立CI/CD流水线
- 一键打包、签名、上传
- **预期效果**：产能提升2-3倍

**6. 马甲化方案升级（优先级：⭐⭐⭐⭐⭐）**
- 研究最新的审核规则
- 升级马甲化技术
- **预期效果**：过审率提升20-30%

**7. 建立知识库（优先级：⭐⭐⭐）**
- 记录所有问题和解决方案
- 建立最佳实践文档
- **预期效果**：减少重复问题，新人上手更快

**8. 团队技能培训（优先级：⭐⭐⭐）**
- 让Flutter开发学习iOS/Android原生
- 让Kotlin开发学习Flutter
- **预期效果**：提升团队灵活性

## 💡 管理建议

### 与主管沟通的技巧

**定期汇报（建议每周一次）：**
- 用数据说话（流水、过审率、产能）
- 主动汇报问题和解决方案
- 管理期望，不要过度承诺
- 及时申请资源支持

**遇到问题时：**
- 不要隐瞒问题
- 带着解决方案去汇报
- 说明需要什么支持
- 给出时间预期（但不要太具体）

### 与团队沟通的技巧

**建立信任：**
- 前期多听少说
- 尊重团队成员的意见
- 承认自己不懂的地方
- 一起解决问题，而不是指责

**明确目标：**
- 让每个人知道自己的职责
- 设定清晰的目标和deadline
- 及时反馈和认可

**处理冲突：**
- 私下沟通，不要公开批评
- 对事不对人
- 寻求双赢方案

## 🎯 第一件事该做什么？

### 第一天上午（最重要的4小时）

**第1件事：与主管深度沟通（2-3小时）**

这是最重要的，必须搞清楚以下问题：

1. **审核问题的严重程度**
   - 现在拒审率是多少？
   - 最近一次过审是什么时候？
   - 如果持续拒审，公司能撑多久？
   - 主管有什么应对方案？

2. **你的KPI和考核标准**
   - 具体的数字目标是什么？
   - 考核周期是多久？
   - 底线是什么？（不能低于多少）

3. **资源和支持**
   - 可以申请什么资源？
   - 遇到问题可以找谁帮忙？
   - 有没有预算？

4. **交接时间表**
   - 什么时候完全交接给你？
   - 交接期间主管会给什么支持？

**第2件事：获取所有必要权限（30分钟）**
- 代码仓库
- 数据看板
- 账号管理系统
- 文档和知识库

**第3件事：查看最近的数据（30分钟）**
- 最近7天的流水数据
- 最近提交的包的过审情况
- 当前的产能数据

### 为什么第一件事是与主管沟通？

1. **明确方向**：避免做无用功
2. **了解紧急程度**：知道哪些问题必须立即解决
3. **建立信任**：让主管知道你在认真对待
4. **获取支持**：明确可以调用的资源

### 第一天下午做什么？

- 与团队成员1对1沟通（每人30分钟）
- 查看代码仓库和文档
- 如果有时间，亲自走一遍打包流程

## ✅ 总结

### 核心原则

1. **先稳住方向1**：这是现金流，不能出问题
2. **快速解决审核问题**：这是当务之急
3. **不要试图同时优化所有方向**：资源有限，要聚焦
4. **建立数据驱动的决策机制**：用数据说话
5. **管理期望**：对上对下都要管理好期望

### 成功的标志

**第1个月：**
- 完全接手方向1
- 审核问题有明显改善
- 团队运转顺畅

**第2-3个月：**
- 产能提升30%以上
- 过审率稳定在XX%以上（根据KPI）
- 成功接手方向2
- 建立了基本的自动化工具

**第3-6个月：**
- 流水持续增长
- 团队效率显著提升
- 方向3重新启动
- 团队成员能力提升
