# 第一周每日行动清单

> 这是一个超级详细的每日todo清单
> 每天照着做就行，不用想太多

---

## 📅 Day 1（第一天）

### 上午（9:00-12:00）

#### ✅ Todo 1：与主管深度沟通（2-3小时）⭐⭐⭐⭐⭐

**具体做什么：**
- [ ] 约主管的时间（至少2小时）
- [ ] 准备一个笔记本或电脑，记录所有信息

**要问的问题（按顺序问）：**

1. **审核问题（最紧急）**
   - [ ] 现在拒审率是多少？
   - [ ] 最近一次过审是什么时候？
   - [ ] 苹果给的拒审理由是什么？（4.3？2.1？）
   - [ ] 如果持续拒审，公司能撑多久？
   - [ ] 主管现在有什么应对方案？

2. **我的KPI**
   - [ ] 我的KPI是什么？具体数字是多少？
   - [ ] 考核周期是多久？（月度？季度？）
   - [ ] 底线是什么？（不能低于多少）

3. **资源和权限**
   - [ ] 我可以申请多少预算？
   - [ ] 我可以招人吗？
   - [ ] 我有什么决策权？

4. **交接时间表**
   - [ ] 什么时候完全交接给我？
   - [ ] 交接期间主管会给我什么支持？

**为什么要做：**
- 这是最重要的，搞清楚方向和期望
- 避免做无用功
- 让主管知道你很认真

---

### 下午（14:00-18:00）

#### ✅ Todo 2：获取所有必要权限（30分钟）

**具体做什么：**
- [ ] 代码仓库访问权限（GitLab/GitHub）
- [ ] 数据看板访问权限（如果有的话）
- [ ] 账号管理系统访问权限
- [ ] 文档和知识库访问权限
- [ ] 测试一下能不能访问

**为什么要做：**
- 没有权限什么都做不了
- 第一天就要搞定

---

#### ✅ Todo 3：拉数据，建立初步认知（1小时）

**具体做什么：**
- [ ] 如果有数据看板，截图保存
- [ ] 拉取最近30天的关键数据：
  - 流水数据（日/周/月）
  - 过审率/拒审率
  - 产能数据（每周产出多少包）
- [ ] 用Excel或笔记记录下来
- [ ] 找出最明显的3个问题

**为什么要做：**
- 快速建立对业务的感知
- 用数据说话

---

#### ✅ Todo 4：团队见面会（1小时）

**具体做什么：**
- [ ] 召集所有团队成员开会（不是1对1）
- [ ] 会议议程：
  1. 自我介绍（5分钟）
  2. 说明我们的目标（10分钟）
  3. 说明工作方式（10分钟）
  4. 听大家的想法（35分钟）

**要说的话（参考）：**
```
大家好，我是XXX，从今天开始负责这个团队。

我们的目标是：稳住iOS流水，提升产能，解决审核问题。

我的工作方式：
- 每天早上10点站会，15分钟
- 遇到问题随时找我
- 我会尽量帮大家解决问题

现在想听听大家的想法，有什么问题或建议都可以说。
```

**为什么要做：**
- 快速建立团队感
- 让大家知道你的风格

---

#### ✅ Todo 5：快速了解代码和文档（1小时）

**具体做什么：**
- [ ] 打开代码仓库，看一下目录结构
- [ ] 找到README或文档，快速浏览
- [ ] 不要深入看代码，只是了解大概

**为什么要做：**
- 建立初步印象
- 知道代码在哪里

---

### 晚上（可选）

#### ✅ Todo 6：整理今天的信息（30分钟）

**具体做什么：**
- [ ] 把今天收集的信息整理成文档
- [ ] 标注出最紧急的3个问题
- [ ] 想一下明天要做什么

## 📅 Day 2（第二天）

### 上午（9:00-12:00）

#### ✅ Todo 1：每日站会（15分钟）⭐⭐⭐⭐⭐

**具体做什么：**
- [ ] 早上10点，召集团队站会
- [ ] 每个人说：昨天做了什么？今天做什么？有什么问题？
- [ ] 你来决策：这个问题怎么解决？

**为什么要做：**
- 建立团队节奏
- 快速响应问题

---

#### ✅ Todo 2：让Flutter开发A演示iOS包制作流程（2小时）

**具体做什么：**
- [ ] 找Flutter开发A（最强的那个）
- [ ] 让他从头到尾演示一遍iOS包的制作流程
- [ ] 你要做的：
  - **看**：每个步骤是怎么做的
  - **问**：为什么这样做？有没有更好的方式？
  - **记**：哪个环节最慢？哪个环节最容易出错？

**要记录的信息：**
- [ ] 第1步：XXX（需要多长时间？）
- [ ] 第2步：XXX（需要多长时间？）
- [ ] ...
- [ ] 最慢的环节是什么？
- [ ] 最容易出错的环节是什么？

**为什么要做：**
- 快速理解业务流程
- 找到瓶颈

---

### 下午（14:00-18:00）

#### ✅ Todo 3：亲自走一遍iOS包制作流程（2-3小时）⭐⭐⭐⭐

**具体做什么：**
- [ ] 在Flutter开发A的指导下，亲自做一遍
- [ ] 从头到尾完整走一遍
- [ ] 记录每个步骤的耗时
- [ ] 记录每个步骤的痛点

**为什么要做：**
- 只有亲自做过，才知道哪里慢、哪里容易出错
- 建立团队信任（团队会觉得"这个主管懂我们在做什么"）

**⚠️ 注意：走完一遍就够了，不要陷入细节！**

---

#### ✅ Todo 4：画出流程图和瓶颈分析（1小时）

**具体做什么：**
- [ ] 用纸或电脑画出完整的流程图
- [ ] 标注每个环节的耗时
- [ ] 标注每个环节的问题
- [ ] **识别Top 3瓶颈**

**为什么要做：**
- 可视化流程
- 找到优化点

---

#### ✅ Todo 2：制定30天行动计划（2-3小时）⭐⭐⭐⭐⭐

**具体做什么：**
- [ ] 基于前3天的了解，制定一个30天行动计划
- [ ] 用Excel或文档记录

**计划内容：**

1. **明确目标**
   - [ ] 30天后要达成什么结果？（用数据说话）
   - [ ] 例如：过审率从X%提升到Y%
   - [ ] 例如：产能从每周X个包提升到Y个包

2. **识别Top 3问题**
   - [ ] 问题1：XXX（影响有多大？）
   - [ ] 问题2：XXX（影响有多大？）
   - [ ] 问题3：XXX（影响有多大？）

3. **制定解决方案**
   - [ ] 针对问题1，解决方案是什么？谁来负责？什么时候完成？
   - [ ] 针对问题2，解决方案是什么？谁来负责？什么时候完成？
   - [ ] 针对问题3，解决方案是什么？谁来负责？什么时候完成？

4. **设定里程碑**
   - [ ] Week 1要完成什么？
   - [ ] Week 2要完成什么？
   - [ ] Week 3要完成什么？
   - [ ] Week 4要完成什么？

**为什么要做：**
- 明确方向
- 让团队知道我们要做什么

**汇报内容：**

**为什么要做：**
- 确保方向正确
- 获得资源支持
- 管理期望
