# Flutter开发工程师精简版面试题-莫桂

候选人：莫桂  
岗位方向：Flutter开发工程师（偏 Android / Flutter 架构）  
经验年限：多年移动端研发经验

## 一、候选人背景摘要

莫桂的简历有几个很鲜明的标签：
1. Android 原生基础较强，长期做过 Java / Kotlin、架构重构、模块化、组件化；
2. Flutter 不是单纯页面开发，而是参与过从 0 到 1 引入、技术选型、规范落地；
3. 做过混合架构、add-to-app、小程序平台、PlatformChannel、JS SDK 兼容等偏复杂场景；
4. 有工程治理意识，涉及 CI/CD、Code Review、模块治理、构建效率；
5. 做过智能硬件、MIDI、WebSocket 实时通信、海外上架合规等较容易追问出深度的项目。

这类候选人的重点不是只问 Flutter 页面开发，而是要验证：
- Flutter 深度到底有多深；
- 架构和工程治理是不是他真实主导；
- Android 原生和混合架构能力是否扎实；
- “从 0 到 1 引入 Flutter”到底做到什么程度；
- AI 赋能研发是不是停留在概念层。

## 二、核心提问清单（建议现场直接使用）

### 1. 你从 Android 原生转到 Flutter 的关键节点是什么？为什么会主导 Flutter 从 0 到 1 引入？
重点看：技术演进路径、选型判断力、是否真的做过技术引入。

### 2. 你在库客音乐提到主导 Flutter 技术栈从 0 到 1 落地。这个“0 到 1”你具体做了哪些事？
重点看：是否真的做过调研、选型、规范、架构搭建，而不是只参与项目开发。

### 3. 你在 Bitsky 做 Flutter + Native 小程序混合架构。请把 add-to-app、engine 复用、PlatformChannel、JS SDK 兼容这套链路完整讲一遍。
重点看：混合架构深度、跨端通信理解、是否有真实复杂项目经验。

### 4. Bitsky 里为什么一定要做 engine 复用？如果不复用，会出现什么问题？
重点看：对 Flutter engine 生命周期、性能和内存开销的理解。

### 5. 你在丰云行里推动现代 Android 架构转型，从 MVP 到 MVVM、从 support-library 到 androidx、从 Java 混编到 Kotlin First。这里面最难的决策是什么？
重点看：架构重构能力、工程治理能力、是否能在大项目里推动变更。

### 6. 丰云行里你提到多工程多 module 治理。为什么单工程多 module 不够？你们具体怎么拆，收益是什么？
重点看：模块化治理的深度，而不是只会说概念。

### 7. KUKEY 智能钢琴项目里，你做了首页长图随屏滚动优化和 MIDI 监听抽象。请各讲一个最能体现你技术能力的细节。
重点看：性能优化能力、复杂业务抽象能力、是否真正动手解决问题。

### 8. 你做过教师端和学生端基于 websocket 的双向实时通信。这个课堂状态同步具体怎么设计？客户端要处理哪些稳定性问题？
重点看：实时通信理解、状态同步设计、线上复杂问题经验。

### 9. 唯衣宝里你主导小票打印组件设计。请讲一下你当时为什么要做分层打印架构，以及如何兼容蓝牙、串口、多模板、多设备。
重点看：抽象能力、硬件相关经验、组件化设计能力。

### 10. 你提到熟悉 Flutter 渲染流程、状态管理、原生通信机制。请你讲一个 Flutter 项目里最能证明你理解底层机制的真实案例。
重点看：Flutter 深度是否足够，不只是业务开发。

### 11. 你在塔影科技提到 Coding AI 赋能 Native 和 Flutter 研发。你们具体把 AI 用在了哪些环节？哪些环节你认为 AI 不能替代人？
重点看：AI 使用成熟度、工程落地能力、边界感。

### 12. 如果现在让你带团队做一个新的 Flutter + Native 混合项目，你会怎么定技术基线、架构边界和工程规范？
重点看：主程/架构能力、方法论完整度、可落地性。
